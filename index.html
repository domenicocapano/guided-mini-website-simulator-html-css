Simulatore didattico: progettazione e sviluppo di un mini-sito web in HTML e CSS

Percorso introduttivo di Informatica – Progettazione di mini siti web in HTML e CSS

Ambiente guidato per la realizzazione di un mini-sito web:
lo studente viene accompagnato passo passo nella progettazione, scrittura e verifica del codice HTML e CSS.

1. Obiettivo del simulatore

Questo simulatore aiuta a progettare un mini-sito dedicato a uno Stato del mondo. Alla fine lo studente dovrà ottenere una cartella contenente:

index.html geografia.html economia.html cultura.html CSS incorporato nei tag <style> cartella immagini

Apri esempio completo: Giappone livello 10

L’esempio completo serve come modello di qualità: osserva struttura, menu, pagine aggiuntive, immagini e crediti, ma non copiare i contenuti.
Prima regola: i nomi dei file devono essere scritti esattamente come indicato. Se il link contiene geografia.html, il file deve chiamarsi proprio geografia.html. Il menu con i pulsanti Home, Geografia, Economia e Cultura deve essere presente in tutte le pagine del sito.

2. Consegna del lavoro e uso degli LLM

Il lavoro prodotto deve essere consegnato secondo le indicazioni del docente, ad esempio tramite piattaforma didattica, cloud, repository o cartella condivisa.

Come consegnare i file

  1. Apri la piattaforma o la cartella indicata dal docente.
  2. Accedi allo spazio del corso o dell'attività.
  3. Apri la consegna, la cartella condivisa o il repository indicato.
  4. Seleziona il comando per allegare o caricare i file.
  5. Carica i file prodotti: index.html, geografia.html, economia.html, cultura.html ed eventuali immagini.
  6. Controlla che tutti i file siano presenti prima di completare la consegna.
Puoi usare strumenti di intelligenza artificiale come ChatGPT, Gemini o Claude e puoi consultare il web. Puoi anche copiare parti di codice o testi come base di lavoro, ma devi poi controllarli, adattarli al tuo Stato, capirne il significato e renderli coerenti con il sito. Il docente potrà chiederti di spiegare ciò che hai inserito.

3. Uso corretto di IA e web

Gli strumenti di IA e le pagine web possono aiutarti, ma il sito finale deve essere ordinato, comprensibile e adatto alla consegna scolastica. Non basta incollare contenuti: devi selezionare le informazioni utili e inserirle in modo chiaro.

Prompt utili per ChatGPT, Gemini o Claude

  • "Spiegami in modo semplice la geografia del/della [Stato], adatta a studenti in fase iniziale di apprendimento di HTML e CSS."
  • "Riassumi l'economia del/della [Stato] in due paragrafi chiari, senza linguaggio troppo difficile."
  • "Correggi questo codice HTML e spiegami l'errore: [incolla codice]."
  • "Trasforma queste informazioni in un testo originale e comprensibile per una pagina web scolastica."

Regola di controllo

Dopo aver usato IA o web, verifica sempre tre aspetti: il testo è comprensibile, le informazioni sono plausibili, il codice funziona nel browser.

4. Scelta dello stile grafico

Per migliorare l'aspetto del sito puoi scegliere uno stile coerente con lo Stato studiato. Il simulatore genera un CSS di base, ma puoi modificarlo cambiando colori, bordi, spaziature e immagini.

  • Stile istituzionale: blu, bianco, grigio; adatto a ricerche ordinate e sobrie.
  • Stile natura: verde, azzurro, beige; adatto a Stati con paesaggi, foreste, mare o montagne.
  • Stile caldo: rosso, arancione, giallo; adatto a Stati con forte identità culturale o climatica.

Attenzione: i colori devono rendere il testo leggibile. Evita sfondi troppo scuri con testo poco contrastato.

7. Pagine aggiuntive facoltative

Il sito minimo contiene quattro pagine. È possibile però aumentare il numero di pagine, se il docente lo richiede o se lo studente vuole approfondire.

  • storia.html - cenni storici principali.
  • turismo.html - luoghi da visitare.
  • curiosita.html - aspetti particolari dello Stato.
  • sport.html - sport diffusi e personaggi importanti.

Ogni nuova pagina deve avere lo stesso menu delle altre pagine, altrimenti il sito diventa difficile da navigare.

8. CSS personalizzato per ogni studente

Ogni studente può avere un CSS diverso scegliendo uno stile dal menu a tendina nella sezione "Inserisci i dati del tuo Stato": blu istituzionale, verde natura, caldo, scuro, mare, deserto, tecnologico oppure Altro CSS personalizzato. Il CSS viene inserito direttamente dentro ogni pagina, tra i tag <style> e </style>, così non serve creare un file CSS separato.

Il CSS va scritto nella parte <head> della pagina HTML, non dentro il <body>.

9. CSS personalizzato: uso guidato

La voce Altro (CSS personalizzato) permette allo studente di progettare un proprio foglio di stile. È utile per chi vuole andare oltre gli stili predefiniti.

Regole minime da rispettare:

  • il testo deve rimanere leggibile;
  • il menu deve restare in alto e visibile;
  • le immagini non devono uscire dalla pagina;
  • il footer con il copyright deve rimanere presente.

10. Passaggi da seguire con il programma Blocco note

1
Prima crea la cartella del progetto.
Per esempio avente nome: Giappone, Francia, Brasile. Dentro questa cartella creerai tutti i file.
2
Crea la sottocartella immagini.
Dentro la cartella creata: crea una cartella chiamata esattamente immagini. Qui metterai bandiera, mappa ed eventuali immagini culturali.
3
Apri Blocco note.
Scrivi o copia il codice generato dal simulatore. Quando salvi, scegli File > Salva con nome.
Non usare Word: aggiunge formattazioni non adatte al codice HTML.
4
Salva correttamente.
Nel campo "Nome file" scrivi, ad esempio, index.html. Nel campo "Salva come" scegli Tutti i file, non "Documento di testo".
5
Inserisci il menu in tutte le pagine.
I pulsanti Home, Geografia, Economia e Cultura devono comparire in index.html, geografia.html, economia.html e cultura.html.
6
Controlla nel browser.
Dopo ogni modifica salva il file e aggiorna la pagina nel browser con F5. Non andare avanti se la pagina corrente non funziona.

13. Inserisci i dati del tuo Stato

Compila i campi. Il simulatore genera il codice delle quattro pagine con CSS incorporato. Gli esempi possono essere sostituiti dallo studente con parole proprie.

Lo Stato deve essere scelto dallo studente dal menu a tendina. Se lo Stato non è presente, scegliere Altro e inserirlo manualmente.

Il Giappone è stato utilizzato dal docente come esempio guidato durante la spiegazione, quindi non può essere scelto per il progetto. Anche se inserito manualmente, il sistema lo bloccherà.

Il CSS viene incorporato automaticamente tra i tag <style> e </style> delle pagine generate.

CSS personalizzato dello studente

Questo riquadro compare solo se scegli "Altro (CSS personalizzato)". È collocato sotto la scelta dello stile per non spostare i campi principali dello Stato.

Mantieni sempre testo leggibile, menu visibile, immagini proporzionate e footer presente.

Il nome dello Stato viene scelto dal menu a tendina superiore. Se scegli "Altro", questo campo diventa modificabile. Il campo è bloccato quando scegli uno Stato dal menu.

Obbligatorio: questo nome comparirà nel copyright del footer.

14. Come costruire le altre pagine

Dopo avere creato index.html, le altre pagine si costruiscono riutilizzando la stessa struttura generale: intestazione, menu in alto, contenuto centrale e piè di pagina.

Metodo consigliato con il Blocco note

  1. Apri il file index.html.
  2. Salvalo con un nuovo nome, per esempio geografia.html.
  3. Cambia il titolo della pagina dentro <title> e dentro <h1>.
  4. Lascia invariato il menu, perché deve comparire in tutte le pagine.
  5. Sostituisci il contenuto centrale con i paragrafi richiesti per quella pagina.
  6. Ripeti lo stesso procedimento per economia.html e cultura.html.
Il menu deve rimanere uguale in tutte le pagine. Cambia solo il contenuto centrale: geografia, economia o cultura.

15. Dove collocare il CSS nelle pagine HTML

Per semplificare il lavoro con il Blocco note, in questa attività il CSS viene inserito direttamente dentro ogni file HTML, nella parte <head>.

La struttura corretta è questa:

<head>
  <meta charset="UTF-8">
  <title>Giappone</title>

  <style>
    body {
      font-family: Arial, sans-serif;
      background-color: #f4f7fb;
    }
  </style>
</head>
Il CSS scritto tra <style> e </style> controlla colori, spazi, bordi, pulsanti e disposizione del menu.

16. Come si realizza il menu con i CSS

Il menu deve stare in alto in tutte le pagine. Non deve essere un elenco puntato. Si usano collegamenti HTML <a>, ma il CSS li trasforma graficamente in pulsanti.

Codice HTML del menu:

<nav class="menu">
  <a class="menu-button" href="index.html">Home</a>
  <a class="menu-button" href="geografia.html">Geografia</a>
  <a class="menu-button" href="economia.html">Economia</a>
  <a class="menu-button" href="cultura.html">Cultura</a>
</nav>

CSS essenziale del menu:

.menu {
  display: flex;
  justify-content: center;
  gap: 8px;
}

.menu-button {
  background-color: #2f75b5;
  color: white;
  text-decoration: none;
  padding: 10px 16px;
  border-radius: 8px;
  font-weight: bold;
}

.menu-button:hover {
  background-color: #163b5c;
  transform: scale(1.06);
}

17. Copyright obbligatorio nel footer

Ogni pagina del sito deve contenere nel footer il copyright con il Nome e Cognome dello studente.

<footer>
  <p>Mini-sito realizzato con HTML e CSS.</p>
  <p>&copy; Mario Rossi - Progetto didattico HTML e CSS</p>
</footer>
Sostituisci "Mario Rossi" con il tuo Nome e Cognome. Il copyright personale è obbligatorio.

19. Anteprima rapida della Home

L'anteprima serve a controllare l'aspetto generale della pagina selezionata. I link funzioneranno correttamente quando i file saranno salvati tutti nella stessa cartella.

20. Se non funziona

  • Se il file non si apre, controlla che non sia stato salvato come index.html.txt.
  • Se un link non funziona, controlla che il nome scritto in href sia identico al nome del file.
  • Se l'immagine non appare, controlla che il file sia nella cartella immagini e che il nome sia scritto correttamente.
  • Se il CSS non viene applicato, controlla che il blocco <style> ... </style> sia dentro il tag <head> della pagina.

21. Trucco didattico: lavorare per micro-obiettivi

La miglioria didattica più importante è trasformare il simulatore in un percorso a stati di avanzamento. Lo studente non dovrebbe solo generare codice, ma vedere chiaramente a che punto si trova: struttura creata, menu funzionante, CSS applicato, immagini controllate, contenuti completati, consegna pronta.

In questo modo il software diventa un vero simulatore didattico: non sostituisce il lavoro dello studente, ma lo guida con piccoli obiettivi verificabili. Il docente può chiedere di consegnare ogni giorno lo stato raggiunto, evitando che gli studenti arrivino alla fine con un sito copiato ma non compreso.
